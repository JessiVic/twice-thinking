## [2.1.4] - 2026-02-01

### Changed
- **成员描述精简优化**：提取核心特质，减少冗余细节
  - 将每位成员的描述从"擅长领域 + 思维特点 + 适用场景 + 特殊身份 + MBTI"重构为"核心特质 + 思维方式 + 擅长场景 + 身份标签"
  - 移除过多的具体例子和背景故事，保留关键特质和思维模式
  - 精简MBTI描述，仅保留类型标签
  - 整合"特殊身份"为简洁的"身份标签"
  - 优化表达方式，使特质描述更加直接和可操作

### Added
- **TWICE核心精神**：在概述部分新增TWICE的核心精神
  - 忠于自我表达
  - 长期主义与扎实基础
  - 保持初心
  - 多元化与可能性

### Technical
- 文档从约1000行精简至约1000行（保持结构完整性的同时提升可读性）
- 成员描述更聚焦于"如何思考"而非"具体事例"
- 便于模型快速理解和应用成员特质

## [2.1.2] - 2026-01-30

### Changed
- **用户主动要求其他成员发言时的节奏控制**：
  - 明确规定：即使用户说"想听其他成员的意见"，也必须遵守1-3人限制
  - 采用轮换机制：理解为"接下来几轮优先让之前较少发言的成员参与"，而不是一次性让所有人说话
  - 支持点名：用户可以具体指定想听哪几位成员，但仍遵守1-3人限制
  - 保持对话节奏：分批轮换，避免信息过载和长篇大论

### Fixed
- 修复了用户主动要求其他成员发言时，系统会让所有成员一次性输出导致对话失控的问题
- 确保对话始终符合真实对话场景，每轮对话长度可控

### Documentation
- 在"对话节奏"部分新增"用户主动要求其他成员发言时"的详细规则
- 明确说明轮换机制和点名支持的具体实施方式

## [2.1.0] - 2026-01-29

### Changed
- **对话节奏优化**：基于用户反馈优化对话交互机制
  - 首轮开场：保持1位成员开场，建立对话基础
  - 后续轮次：1-3位成员参与（不超过3位），避免信息过载
  - 每位成员每次只问一个最重要的问题，保证对话质量和思考深度
  - 新成员可以随时加入，围绕同一核心话题，从不同角度切入
  - 避免过度深挖，适度提问，不要把问题问得太细、太深导致偏题

### Added
- **元认知与方向调整机制**：
  - 成员能觉察对话状态（是否偏题、太深、太细）
  - 温和提醒并征求用户意见，而不是直接转向
  - 提供选项让用户选择继续当前方向或换个角度
  - 尊重用户决定，最终由用户决定对话方向
  - 示例表达：
    - "我觉得我们好像问得有点深了，你觉得呢？"
    - "要不要换个角度看这个问题？"
    - "我们可以继续深入，也可以先看看大方向，你更想聊哪个？"
- **CHANGELOG.md**：独立的版本变更记录文件

### Documentation
- 更新 `SKILL.md` 工作流程部分，详细说明对话节奏控制
- 新增"元认知与方向调整"章节，指导成员如何温和引导对话方向
- 更新"注意事项"部分，强调"一次一个问题"和"深度优于广度"原则
- 从SKILL.md分离版本历史到CHANGELOG.md

## [2.0.0] - 2026-01-25

### Changed - 对话机制全面升级
- **[Breaking] 从"一次性输出"到"逐轮对话"**：
  - 旧模式：一次性输出所有成员的完整分析和建议
  - 新模式：逐轮对话，每轮1-2位成员发言，等待用户回复后再继续
  - 核心理念：对话式交流，而非单向咨询
  - 朋友式对话：像朋友聊天一样自然真实，而非咨询师式的客套和说教

### Added
- **深挖提问技巧**：三层提问法（表层、中层、深层）
- **挑战性提问**：挑战假设、指出矛盾、质疑"应该"、反问动机、揭示盲区
- **开场方式优化**：简短温暖的问候，直接进入对话，不做冗长介绍
- **灵活总结时机**：根据用户需要灵活调整，不固定在对话结束时才总结

### Documentation
- 新增"核心理念：平衡支持与挑战"章节
- 新增"深挖提问技巧：三层提问法"章节
- 新增6个详细的对话示例（职业决策、创意瓶颈、个人成长、个性坚持、纠结行动、面对挫折）
